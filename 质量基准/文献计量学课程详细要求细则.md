### Lesson 3-4：数据获取

#### 核心知识点

1. **数据源选择**

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| Web of Science | 质量高、字段全、引文完整 | 学术研究首选 |
| Scopus | 覆盖广、欧洲文献多 | 补充检索 |
| CNKI | 中文文献 | 国内研究 |
| Google Scholar | 覆盖最广、但质量参差 | 初步探索 |
| OpenAlex | 免费、开放、API 友好 | 大规模分析 |

2. **检索式设计原则**

```
三层结构：
├── 对象词（研究主题）
│   └── 核心概念 + 变体 + 同义词
├── 场景词（应用领域）
│   └── 限定范围的关键词
└── 排除词（噪声过滤）
    └── 排除无关领域的术语
```

3. **检索式迭代优化**
   - v0：初步检索，评估查全率
   - v1：调整关键词，平衡查全与查准
   - v2-final：最终版本，记录变更理由

4. **数据导出规范**
   - 格式：Plain Text（全记录 + 参考文献）
   - 字段：至少包含 PT, AU, TI, SO, DE, ID, AB, C1, CR, PY, TC, DI, UT
   - 编码：UTF-8

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 掌握 | WoS 高级检索语法 |
| 应用 | 能设计三层结构检索式 |
| 产出 | 检索式设计文档（含迭代记录） |
| 文档 | `config/query.yaml` 或类似配置文件 |

### Lesson 5-6：分析方法

#### 核心知识点

1. **关键词共现分析**
   - 原理：两关键词同时出现在同一文献中，则存在共现关系
   - 矩阵：W = K.T @ K（K 为文献-关键词矩阵）
   - 用途：识别研究热点、主题聚类

2. **共被引分析**
   - 原理：两篇文献同时被第三篇文献引用，则存在共被引关系
   - 矩阵：C = A.T @ A（A 为文献-参考文献矩阵）
   - 用途：揭示知识基础、学科结构

3. **文献耦合分析**
   - 原理：两篇文献引用了相同的参考文献，则存在耦合关系
   - 矩阵：B = A @ A.T
   - 用途：发现研究前沿、相似文献

4. **合作网络分析**
   - 原理：两作者/机构共同发表文献，则存在合作关系
   - 用途：识别核心团队、合作格局

5. **聚类方法**
   - LLR 聚类：基于对数似然比，适合关键词聚类
   - 谱聚类：基于特征值分解
   - 模块度优化：Newman-Girvan 贪婪算法

6. **突现检测（Burst Detection）**
   - 算法：Kleinberg 突现检测
   - 参数：s（状态数）、γ（控制参数）
   - 用途：识别新兴趋势、热点变迁

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 理解 | 四种网络分析的原理与适用场景 |
| 掌握 | CiteSpace 参数设置 |
| 应用 | 能解读聚类结果与突现图谱 |
| 产出 | 网络图谱 + 解读文档 |

### Lesson 7-8：网络指标

#### 核心知识点

1. **节点级指标**

| 指标 | 含义 | 计算公式 |
|------|------|----------|
| Degree（度） | 节点的连接数 | d(v) = \|N(v)\| |
| Weighted Degree（加权度） | 考虑边权的度数 | d_w(v) = Σw(e) |
| Betweenness（中介中心性） | 经过该节点的最短路径比例 | CB(v) = Σσst(v)/σst |
| PageRank | 网页排名思想，考虑邻居重要性 | PR(v) = (1-d) + d×Σ(PR(u)/L(u)) |
| Closeness（接近中心性） | 到其他节点的平均距离倒数 | CC(v) = (n-1)/Σd(v,u) |
| Eigenvector（特征向量中心性） | 邻居重要性的加权和 | Ce(v) = 1/λ × ΣA(v,u)Ce(u) |

2. **图级指标**

| 指标 | 含义 | 解读 |
|------|------|------|
| Nodes/Edges | 节点数/边数 | 网络规模 |
| Density（密度） | 实际边数/可能边数 | 网络紧密程度 |
| Components | 连通分量数 | 网络连通性 |
| Largest Component Ratio | 最大连通分量占比 | 主流群体覆盖 |
| Communities | 社团数 | 主题/团队分化程度 |
| Modularity（模块度） | 社团结构强度 | 0.3-0.7 为合理范围 |
| Avg Clustering Coefficient | 平均聚类系数 | 局部聚集程度 |
| Diameter | 直径（最大连通分量） | 网络最远距离 |
| Avg Path Length | 平均路径长度 | 信息传播效率 |

3. **聚类评价指标**
   - Q 值（Modularity）：> 0.3 表示聚类结构显著
   - S 值（Silhouette）：> 0.5 表示聚类合理，> 0.7 为高可信

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 理解 | 各指标的数学含义与适用场景 |
| 掌握 | 能计算并解读网络指标 |
| 应用 | 识别核心节点、关键文献 |
| 产出 | `network_metrics_*.csv` + `network_qc_summary.csv` |

### Lesson 17-18：案例讲解（五步法框架）

#### 五步法框架

```
Step 1: 问题定义
    ├── 明确研究问题
    ├── 确定分析目标
    └── 选择分析方法

Step 2: 数据获取
    ├── 选择数据源
    ├── 设计检索式
    ├── 导出数据
    └── 数据清洗

Step 3: 网络构建
    ├── 构建共现/共被引/耦合矩阵
    ├── 网络剪枝（可选）
    └── 参数设置

Step 4: 指标计算
    ├── 节点级指标
    ├── 图级指标
    └── 聚类评价

Step 5: 结果解读
    ├── 图谱解读
    ├── 指标分析
    ├── 结论提炼
    └── 报告撰写
```

#### 案例分析要点

1. **检索式设计**
   - 记录迭代过程
   - 说明变更理由
   - 最终检索式可复现

2. **数据清洗**
   - 去重方法（UT/DOI）
   - 筛选标准（文献类型、语言、相关性）
   - PRISMA 流程图

3. **参数固化**
   - 时间切片设置
   - 节点选择标准（g-index, Top N）
   - 剪枝方法（Pathfinder, MST）
   - 聚类算法（LLR, MI）

4. **结果呈现**
   - 图谱清晰、有图注
   - 指标表格规范
   - 结论有数据支撑

### Lesson 19：深度解读与异常处理（新增）

> 来源：文献计量学课件 Lesson19 & 20.pdf（第1-15页）

#### 核心知识点

1. **时间切片与指标解读**

| 指标 | 含义 | 解读方法 |
|------|------|----------|
| 时间切片（Time Slicing） | 按时间段分割数据 | 观察结构演化，按数据分段 |
| 指标①：发文/被引频次 | 反映增长速度 | 注意"量≠质"，需结合中心性 |
| 指标②：突现强度（Burst Strength） | Kleinberg 算法检测 | 对时间窗口敏感，反映变化速率 |
| 指标③：平均出版年（MPY） | 节点颜色上的年份 | 红色=新，蓝色=旧，标识时间新旧 |

2. **聚类解读三判断**

| 判断方法 | 内容 | 说明 |
|----------|------|------|
| 判断A：时间线图（Timeline View） | 每个聚类的起始、活跃、跃迁时间 | 判断主题生命周期 |
| 判断B：时段对比 | 将数据集分为"早期""近期"等，对比聚类转变 | 识别研究范式转移 |
| 判断C：学科品性（overlay） | 同一聚类在不同学科中的分布 | 识别跨学科特征 |

3. **聚类标签算法对比**

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| LLR | 对数似然比，偏重独特性 | 关键词聚类（推荐） |
| LSI | 潜在语义索引 | 语义相似性 |
| MI | 互信息 | 信息论视角 |

> **原则**：聚类解读 = "活"+"核"。"活"看时间线，"核"看中心性。

4. **结构优势分析（SVA）**

- SVA 原理：一篇研究原论文如果被多个不同聚类引用，则具有很大变革潜力（Transformative Potentials）
- 识别方法：高中心性 + 高突现强度的节点
- 用途：识别领域变革性文献（milestone 候选）

5. **异常处理**

| 异常类型 | 表现 | 处理方法 |
|----------|------|----------|
| 外部冲击 | 如 2020 年 COVID-19 导致发文骤降 | 指出异常→可能原因→选择验证方法→验证结果 |
| 数据库记录延迟 | 新发表文献收录滞后 | 对比不同时间窗口验证 |
| 同名异义 | "Transformer"同时指 DL 架构和电力变压器 | 语义筛选，分集分析 |

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 掌握 | 时间切片、突现检测、聚类解读三判断 |
| 应用 | 能从图谱中解读研究热点、知识基础、前沿趋势 |
| 应用 | 能识别并处理异常（外部冲击、记录延迟、同名异义） |
| 产出 | 趋势问题 Markdown（`reports/trend_question.md`） |
| 作业 | 1 张图（发文/突现/overlay 之一）+ 指标解读，200 字 + 基准文献清单 10 篇 |

### Lesson 20：Milestone 识别与技术路线图（新增）

> 来源：文献计量学课件 Lesson19 & 20.pdf（第16-30页）

#### 核心知识点

1. **Milestone 文献的四准则排序**

| 准则 | 含义 | 优势 | 局限 |
|------|------|------|------|
| 准则①：被引次数（Citation Counts） | 反映影响力 | 直观、易获取 | 偏向老文献；综述偏高 |
| 准则②：突现强度（Burst Strength） | 反映某时期研究热度 | 识别新兴关键文献 | 对时间窗口敏感 |
| 准则③：中介中心性（Betweenness Centrality） | 反映桥接不同知识群的能力 | 识别变革性信号 | 计算依赖网络结构 |
| 准则④：Sigma 值 | Burst + Centrality 的加权综合 | CiteSpace 综合指标，sigma 越高越关键 | 需 CiteSpace 计算 |

> **原则**：没有单一指标能完美排序，多准则交叉验证更可靠。

2. **技术路线图（Technology Roadmap）**

| 要素 | 说明 |
|------|------|
| 时间轴 | 过去→现在→未来，标注关键里程碑 |
| 层级轴 | 研究层级（材料→器件→芯片→系统→应用） |
| 颜色/形状 | 不同技术路线、TRL 等级 |
| TRL（技术就绪度） | 1-9 级，从基础研究到产业应用 |
| 关键里程碑 | 标注变革性文献/事件 |

3. **技术路线图绘制要点**

- 从知识基础（共被引网络）→ 应用前沿（关键词突现）→ 前沿方向
- 包含四个维度：
  - 知识基础的演化
  - 应用前沿的走向
  - 核心团队/机构的分布
  - 应用场景/技术维度
- 选择标准：被引次数 + 突现强度 + 中心性，三者交叉验证

4. **学科品性陷阱**

- 同一术语在不同学科中有不同含义（如 "Optical Computing" 在医学指 OCT，在 AI 指光神经网络）
- 应对：在关键阶段用 WoS 学科品性过滤，数据集在进出阶段明确每个聚类的学科归属

5. **M2 提交规范**

| 提交物 | 要求 |
|--------|------|
| 3 图 1 表 | 图1：共被引/关键词聚类图；图2：时间线+突现图；图3：合作/机构网络图 |
| 1 表 | Top 10 milestone 文献选择表（含被引次数、突现强度、中心性、sigma 值） |
| 解读 | 每图 <200 字解读（研究问题、方法、发现、局限） |
| 总述 | 200 字/图，回答：什么研究问题、需要什么方法、发现什么、局限 |

6. **QC Checklist（M2 质量控制清单）**

| 检查项 | 要求 |
|--------|------|
| 图数量 | 是否有 3 张图，每图对应一个不同研究问题（知识基础/前沿/团队） |
| 表数量 | 是否有 1 张表，至少 2 个指标交叉验证 milestone 文献 |
| 解读 | 每图/表是否有 <200 字的解读（问题、方法、发现、局限） |
| 参数记录 | 是否记录了所有分析参数（版本、时间范围、阈值、时间切片等） |
| 偏差说明 | 是否说明了数据来源、筛选标准、潜在偏差 |

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 掌握 | Milestone 四准则排序（被引、突现、中心性、Sigma） |
| 掌握 | 技术路线图绘制方法（时间轴+层级轴+TRL） |
| 应用 | 能从共被引网络识别知识基础，从突现检测识别前沿 |
| 应用 | 能绘制简化技术路线图 |
| 产出 | 3 图 1 表 + 解读（M2 提交物） |
| 自检 | 通过 QC Checklist 全部检查项 |

## 六、常见问题与注意事项

### 6.1 检索式设计

- ❌ 避免：过度依赖具体应用场景词，导致漏检
- ✅ 建议：使用三层结构，平衡查全与查准
- ✅ 建议：记录每次迭代的变更理由

### 6.2 数据清洗

- ❌ 避免：直接使用检索结果，未做去重
- ✅ 建议：UT + DOI 双重校验
- ✅ 建议：明确筛选标准，记录排除原因

### 6.3 参数设置

- ❌ 避免：使用默认参数，未说明选择理由
- ✅ 建议：参数固化文档，可追溯
- ✅ 建议：对比不同参数的结果稳定性

### 6.4 结果解读

- ❌ 避免：只描述现象，不解释原因
- ❌ 避免：结论超出数据支撑范围
- ✅ 建议：结合领域知识解读
- ✅ 建议：指出研究的局限性

### 6.5 可复现性

- ❌ 避免：代码无注释、依赖无版本
- ❌ 避免：操作步骤缺失
- ✅ 建议：使用模板提供的标准结构
- ✅ 建议：README 包含完整运行命令

### 6.6 【L19 新增】异常处理

- ❌ 避免：忽略外部冲击（如疫情、政策变化）对发文趋势的影响
- ❌ 避免：不说明数据库记录延迟
- ✅ 建议：识别异常→可能原因→选择验证方法→验证结果
- ✅ 建议：同名异义词（如 Transformer）必须做语义筛选

### 6.7 【L20 新增】Milestone 识别

- ❌ 避免：仅用被引次数排序（偏向老文献和综述）
- ❌ 避免：忽略 Sigma 值（CiteSpace 综合指标）
- ✅ 建议：四准则交叉验证（被引+突现+中心性+Sigma）
- ✅ 建议：在技术路线图上标注 milestone

### 6.8 【L20 新增】技术路线图

- ❌ 避免：只画时间线，不区分研究层级
- ❌ 避免：不标注 TRL 等级
- ✅ 建议：时间轴 + 层级轴 + 颜色/形状 + 关键里程碑
- ✅ 建议：学科品性陷阱：同一术语在不同学科中含义不同，需在进出阶段明确聚类学科归属

共现矩阵：     W = K.T @ K
共被引矩阵：   C = A.T @ A

中介中心性：   使用 distance = 1/weight
